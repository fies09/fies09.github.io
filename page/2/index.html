测试用例

发表于 2021-06-22 更新于 2022-03-23 分类于 软件测试 阅读次数：
  |   阅读次数

一,

测试用例:指为实施测试而向被测试系统提供的输入数据,操作或各种环境设置以及期望结果的一个特定的集合.

包含内容:ID,功能模块,优先级,用例标题,预置条件,测试数据,执行步骤,预测结果,实际结果

等价类划分:两位数加法程序
输入两位整数,相加后,结果正确显示.要求:
输入的整数值在-99到99之间,包含(-99和99)
非法输入应被拒,并显示错误信息
等价类划分:
是一种最为典型的黑盒测试方法
要对需求说明书中的各项需求,尤其是功能需求进行细致分析
把程序的输入划分成若干个部分,从每个部分中选少数代表性数据作为测试用例

划分等价类划分典型参考:
1,若某个输入条件说明了一个必须成立的情况,则可划分一个有效等价类和一个无效等价类
2,若某个输入条件对取值范围进行了规定,则可划分一个有效等价类和两个无效等价类
3,如果输入条件是一个布尔量,则可以确定一个有效等价类和一个无效等价类
4,若已划分的某等价类中的各元素在程序中的处理方式不同,则应将此等价类进一步划分成更小的等价类
5,若规定输入数据必须遵循的规则,则可以确立一个有效等价类(符合规则)和若干个无效等价类(从不同角度违反规则)
6,若输入条件是集合,且程序会对每个值进行不同的处理,则应为每个输入值确定一个有效等价类,其他数据作为一个无效等价类

设计等价类表
序号,功能项(两位整数相加),有效等价类,编号,无效等价类,编号

设计测试用例
为等价类表中的每一个等价类分配一个唯一的编号
设计一个测试用例,使它能够尽量覆盖的有效等价类.重复该操作,从而使所有有效等价类均被覆盖
设计一个测试用例,使它能够覆盖一个无效等价类,重复该操作,从而使所有无效等价类均被无效覆盖

设计测试用例:(与等价类表相结合)
编号.输入数值,所属等价类,预期输出

①划分等价类
②设计等价类表
③编写等价类测试用例

边界法:
应当选取证号等于,刚刚大于或刚刚小于边界的值作为测试数据
重点测试最后一个肯定合法的数据和刚刚超过边界的非法数据
通常和等价类划分一起使用,产生一套完整的测试用例

软测概念3

发表于 2021-06-22 更新于 2022-03-23 分类于 软件测试 阅读次数：
  |   阅读次数

概念:
为了发现错误而运行程序的过程

软件测试工程师的基本素质:
正确高效的沟通能力
超强责任心
要坚持原则
有较全面的技术知识

必备技能:
测试专业能力
软件编程技能
网络,操作系统,数据库,中间件等知识

缺陷:
在软件工程整个生命周期中任何背离需求,无法正确完成用户所要求的功能的问题,包括存在于组件,设备或系统软件中因异常条件不支持而导致系统的失败都属于缺陷.

产生原因:
需求不明确,本身存在问题
无法收集到用户的所有需求
在软件的需求调研和设计阶段存在各种问题导致用户需求被错误的理解和传递
用户需求随工作或使用环境的变化以及时间的推移也会随之变化
软件技术的发展落后于不断复杂的软件需求

无效缺陷:不被开发人员修改的缺陷
出现原因:
1,测试环境配置错误,测试工程师不仔细检查测试环境,直接当成一个缺陷提交了
2,需求理解错误,对一个概念的理解错误,造成测试方法错误,不讨论就提交缺陷
3,缺陷重复,同一个缺陷被两个测试工程师提交

合法的无效缺陷:
技术上无法修改
修改成本太大,不影响用户使用

重现缺陷的方法
借助于别的工具,如debug或者dump工具去分析,
检查软件测试环境是否有异于开发环境,
检查网络配置是否与客户环境一致
检查被测对象的版本信息,确认测试版本的正确性
状态缺陷是否在特定的软件状态中暴露

缺陷报告:
实例1:
简单描述：字处理过程中，进行插入数字或者插入符号时，系统崩溃。
详细描述：
1.软件测试环境为Windows 2003
2.启动Boffice软件，然后创建新文档
3.插入一行文本，当鼠标变成了漏斗状时，进行插入数字或插入符号任一操作
4.系统崩溃
相关附件：
–附件1：系统崩溃的画面
缺陷原因分析：
–初步分析与自动存盘有关系，因为在鼠标变成漏斗时，软件正在进行自动存盘操作

按测试阶段划分:
单元测试 => 集成测试 => 系统测试
=> Alpha测试 => Beta测试 => 验收测试
