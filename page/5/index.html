1,Scrapy的安装配置(根据个人实际情况,每个人项目不一样)
①,先安装将twisted文件放到E盘根目录下
#前提必须安装wheel模块,为了安装whl文件
②,E:—>pip install Twisted-18.4.0-cp36-cp36m-win_amd64.whl
③,pip install scrapy==1.5.0

2,在E盘创建Scrapy工程:
E: —> scrapy startproject doubanmovie —>
cd doubanmovie —>scrapy genspider moviespider douban.com

3,爬虫主程序的编写:
在Pycharm工具打开doubanmovie下级目录的doubanmovie
①,添加浏览器标识
将rotate_useragent文件放到doubanmovie文件夹下
②,在settings中的DOWNLOAD_MIDDLEWARES中添加配置信息
添加禁用框架自带的浏览器标识及设置浏览器标识
#禁用框架自带的浏览器标识
‘scrapy.contrib.downloadermiddleware.useragent.UserAgentMiddleware’: None,
#设置浏览器标识
‘doubanmovie.rotate_useragent.RotateUserAgentMiddleware’:400
②-①,编写settings,py,启动管道组件,ITEM_PIPELINES 以及其他相关设置

③,编写主程序

④,执行
scrapy crawl (spiders下文件,不包含后缀)

注意:设置浏览器标识时,第一个应为项目名

自动化测试

参考链接

1,自我介绍:
面试官，你好XXX，我叫，19年毕业，从事软件测试将近2年的时间。在此期间做过一些项目也积累过一些经验，能够独立地完成软件测试流程的一个工作。我之前主要做过的是功能测试，web自动化测试、接口测试、也有用过ipref3做过一些性能方面的测试。我对于缺陷管理工具比如禅道，版本控制器git与svn能够熟悉应用,对于app测试也比较熟悉,数据库、linux、Fiddler、postman这些应用也比较熟悉。以上就是我的一些简单的自我介绍，谢谢。

2,web测试和app测试的区别
①Web项目一般都是b/s架构,基于浏览器的,而app则是c/s的,必须有客户端,
②web测试只要更新了服务器端,客户端就会同步更新,而客户端是可以保证每一个用户的客户端完全一致的,而app端不能够保证完全一致,除非用户更新了客户端,这意味着客户端所使用的核心版本必须进行回归测试一遍
③性能方面:web测试只考虑响应时间,而app还需要关心流量,电量,CPU,GPU,Memory这些了
④兼容性:web:浏览器,电脑硬件,
app:phone,pad,分辨率,机型,版本号,系统类型
④相比之下,app还有更多的专项测试:
健壮性测试:异常场景(中断,来电,关机,重启),弱网测试.

安装,卸载,更新:

界面操作,横竖屏切换,多点触控等测试

3,测试报告主要内容:
测试过程和结果
风险评估
缺陷汇报和分析
测试工作总结和改进

4，登陆模块
UI界面模块:输入框和按钮对齐方式是否一致,高度长度是否合适,界面是否正常
功能模块:用户名密码为空点击是否有提示
用户名密码正常是否正常跳转到登录页面
密码是否包含特殊字符,
密码长度是否在范围内
忘记密码.注册以及切换其他账号登录是否正常实现
性能模块:跳转到登录界面是否超出时长,登录成功登录成功时长是否超过5秒
记住密码功能
密码前后出现空格如何处理
兼容性:主流浏览器是否正常显示,不同平台是否可以正常显示,移动设备是否可以正常显示
不同分辨率
可用性:
是否支持全键盘操作,是否有快捷键,
输入用户名密码回车是否可以登录
Tab键是否可以切换输入框

5,postman中post和get的区别
get请求:直接在浏览器里输入就可以,只要在浏览器里面直接请求到的,都是get请求
post请求:需要借助工具来发送
区别:
1,get使用url或cookie传参,而post将数据放到body中
2,get的url参数长度有限制,而post的数据则可以非常大
3,post比get安全,因为数据在地址栏上不可见,
4,一般get请求用来获取数据,post请求用来发送数据

6,给你一个网站如何测试
1,查找需求说明,网站设计等相关文档,分析测试需求
2,制定测试计划,确定测试范围和测试策略
3,设计测试用例

7,为什么要软件测试
以最少的人力,物力和时间找到软件中潜在的各种错误和缺陷,通过修正各种错误和缺陷提高软件质量,回避软件发布后由于潜在的软件缺陷和错误造成的隐患以及带来的商业风险
